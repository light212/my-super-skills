# 移动端 UX 设计

## 核心约束

| 约束 | 影响 | 应对 |
|------|------|------|
| 小屏幕 | 信息有限 | 渐进披露、折叠 |
| 触摸操作 | 精度低 | 大点击区域 |
| 移动场景 | 注意力分散 | 简化流程 |
| 网络不稳定 | 加载慢 | 离线支持 |

---

## 触控设计

### 点击区域

```
最小：44×44px（Apple HIG）
推荐：48×48px（Material Design）

实现：
button {
  min-height: 48px;
  min-width: 48px;
  padding: 12px 16px;
}
```

### 手势设计

| 手势 | 含义 | 场景 |
|------|------|------|
| 点击 | 选择/激活 | 按钮、链接 |
| 长按 | 上下文菜单 | 列表操作 |
| 滑动 | 滚动/切换 | 轮播、列表 |
| 双击 | 放大 | 图片、地图 |

### 触控反馈

```css
.button:active {
  background: rgba(0, 0, 0, 0.1);
  transform: scale(0.98);
}
```

## 导航模式

### 底部导航栏（推荐）

```
优点：
- 单手可达
- 固定位置
- 清晰入口

数量：3-5 个
```

### 汉堡菜单

```
适用：功能多（>5个）

缺点：
- 发现性差
- 点击次数多
```

## 表单设计

### 输入优化

```
键盘类型：
- type="email" → 邮箱键盘
- type="tel" → 数字键盘
- type="search" → 搜索键盘

自动完成：
- autocomplete="name"
- autocomplete="email"
```

### 布局原则

```
✅ 单列布局
❌ 多列表单

分步表单：
- 每步 ≤ 3 个字段
- 显示进度
- 可保存进度
```

## 特殊组件

### 底部动作面板

```
用途：筛选、分享、详情

特性：
- 从底部滑出
- 支持拖拽
- 背景遮罩
```

### Toast 提示

```
特性：
- 2-3秒自动消失
- 不阻断操作
- 底部或居中
```

### 下拉刷新

```
实现：
- 监听 touch 事件
- 下拉 > 阈值触发
- 显示刷新动画
```

## 性能优化

### 图片优化

```
格式：WebP > JPEG/PNG
尺寸：srcset 多尺寸
加载：loading="lazy"
```

### 动画性能

```
GPU 加速：
- transform
- opacity

避免：
- width/height
- top/left
```

### 骨架屏

```
优点：
- 用户感知更快
- 布局稳定
```
