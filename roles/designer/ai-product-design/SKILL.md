# AI 产品设计模式

## 核心挑战

| 挑战 | 传统产品 | AI 产品 | 设计应对 |
|------|----------|---------|----------|
| 输出结果 | 确定性 | 不确定性 | 提供调整空间 |
| 响应时间 | 即时 | 需要等待 | 流式输出 |
| 错误类型 | 系统错误 | 幻觉/偏见 | 来源标注 |
| 用户预期 | 精确匹配 | 需调整预期 | 明确能力边界 |

---

## 八大设计模式

### 模式 1：渐进式披露

**问题**：AI 输出可能很长，一次性显示信息过载

**解决方案**：
```
第一层：简短摘要/结论
第二层：详细内容（点击展开）
第三层：原始数据/来源（按需查看）
```

**应用**：
- 模型推荐
- AI 回答内容
- 复杂分析结果

---

### 模式 2：置信度展示

**问题**：AI 输出不确定，用户不知道可信度

**解决方案**：
```
高置信度：✅ 确认标记
中置信度：⚠️ 需验证标记
低置信度：❓ 建议核实标记
```

**应用**：
- 内容审核
- 智能分类
- 自动标注

---

### 模式 3：流式输出

**问题**：AI 生成长内容时，用户等待体验差

**解决方案**：
```
逐字/逐段显示 → 实时反馈

优点：
- 用户感知响应快
- 可以提前判断是否需要
- 心理等待感降低
```

**应用**：
- AI 对话
- 长文生成
- 代码生成

---

### 模式 4：来源标注

**问题**：AI 可能幻觉，用户无法验证

**解决方案**：
```
AI 回答 + 来源链接

设计要点：
- 来源可点击
- 显示来源可信度
- 多来源时显示所有
```

**应用**：
- AI 搜索
- 知识问答
- 研究助手

---

### 模式 5：预期管理

**问题**：用户对 AI 能力预期过高

**解决方案**：
```
明确能力边界：
- 功能描述：实际能力，不过度承诺
- 错误处理：友好提示，引导正确使用
- 引导示例：展示最佳使用方式
```

**应用**：
- 产品介绍页
- 功能引导
- 错误提示

---

### 模式 6：多模型对比

**问题**：用户不知道选哪个模型

**解决方案**：
```
场景驱动 + 对比展示

流程：
1. 场景入口："你想做什么？"
2. 推荐模型：首选/备选/性价比
3. 详细对比：表格 + 雷达图
4. 决策建议：哪个更适合你
```

**应用**：
- 模型选择
- AI 工具对比
- 方案选择

---

### 模式 7：迭代优化

**问题**：AI 第一次输出可能不满意

**解决方案**：
```
提供调整能力：
- 🔄 重新生成
- ⚙️ 参数调节
- 👍👎 反馈
- 📝 编辑输出
```

**应用**：
- 文本生成
- 图像生成
- 代码生成

---

### 模式 8：人机协作

**问题**：完全自动化可能出错

**解决方案**：
```
AI 建议 + 人工确认

流程：
AI：检测到 5 个疑似问题
用户：审核确认
AI：执行修复

设计要点：
- AI 提供建议，不直接执行
- 用户有最终决定权
- 提供撤销能力
```

**应用**：
- 数据处理
- 内容审核
- 自动化任务

## 状态设计规范

### 加载状态

```
生成中：
┌─────────────────────────────┐
│ 🤖 AI 正在思考...            │
│ ▓▓▓▓▓▓░░░░ 60%              │
└─────────────────────────────┘

流式输出：
┌─────────────────────────────┐
│ AI：这是生成的第一段内容...█  │
│    (光标闪烁)                │
└─────────────────────────────┘
```

### 错误状态

```
┌─────────────────────────────┐
│ ❌ 生成失败                  │
│                             │
│ 可能原因：                   │
│ - 输入过长                   │
│ - 网络问题                   │
│                             │
│ [重试] [修改输入]            │
└─────────────────────────────┘
```

### 空状态

```
┌─────────────────────────────┐
│ 🤖                           │
│ 试着问我一些问题             │
│                             │
│ 示例：                       │
│ - 帮我写一封邮件             │
│ - 解释什么是机器学习         │
└─────────────────────────────┘
```

## 交互原则

### 响应性

| 场景 | 传统产品 | AI 产品 |
|------|----------|---------|
| 点击按钮 | 即时反馈 | 流式反馈 |
| 长任务 | 进度条 | 逐字显示 |
| 错误 | 重试 | 重试 + 修改输入 |

### 可预测性

```
让用户知道 AI 会做什么：
- 功能描述明确
- 示例展示
- 能力边界说明
```

### 可控性

```
用户可以控制：
- 停止生成
- 重新生成
- 修改参数
- 编辑输出
- 提供反馈
```
