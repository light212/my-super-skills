# UX 设计

## 用户旅程
- 绘制完整用户路径：触达 → 注册 → 首次使用 → 核心操作 → 留存
- 标注每个节点的用户情绪和痛点
- 找到关键转化点，优化体验

## 信息架构 (IA)
- 内容分组遵循用户心智模型，不是技术结构
- 导航层级不超过 3 层
- 命名用用户语言，不用技术术语

## 交互设计 (IxD)
- 操作反馈即时可见（点击、加载、成功、失败）
- 减少用户决策负担：合理默认值、渐进式披露
- 关键操作可撤销

## 可用性启发（Nielsen 10）
1. 系统状态可见
2. 匹配现实世界
3. 用户控制和自由
4. 一致性和标准
5. 错误预防
6. 识别而非回忆
7. 灵活性和效率
8. 美学和极简
9. 帮助用户识别和恢复错误
10. 帮助和文档

## 无障碍 (WCAG)
- 颜色对比度 ≥ 4.5:1（正文）、≥ 3:1（大文本）
- 所有交互元素可键盘操作
- 图片有 alt 文本，图标有 aria-label
- 不依赖单一感官传达信息
